# 全生命周期标准化技能体系 (SDLC SKILL System) - 检索与调用矩阵

本指南用于总览并按多维度快速检索软件开发生命周期（SDLC）全流程的标准化技能（SKILL）。整套体系旨在提升研发质量、降低风险，并支持根据具体项目特性进行定制化调用。

## 📦 技能库一览 (SKILL Repository)

该体系被拆分为 4 个核心的 Trae 智能 SKILL，您可以通过以下命令或自然语言请求直接调用对应的智能体能力：

1. **研发阶段** 👉 `sdlc-dev` (包含: 需求分析、架构设计、技术选型、编码规范、代码审查)
2. **测试阶段** 👉 `sdlc-test` (包含: 单元测试、集成测试、自动化框架搭建、缺陷管理、性能安全测试)
3. **集成阶段** 👉 `sdlc-ci-cd` (包含: CI/CD流水线、版本控制、容器化与微服务、依赖与环境管理)
4. **发布运维阶段** 👉 `sdlc-ops` (包含: 发布策略、灰度与回滚、监控告警、日志管理、容量与灾备)

## 🔍 多维度组合调用指南

通过向AI助手输入包含以下维度的**自然语言指令**，AI将自动提取对应阶段的技能，并基于您的“上下文约束”输出量身定制的最佳实践。

### 维度一：按项目类型 (Project Type)
| 项目类型 | 核心关注点 | 典型调用示例 | 触发技能 |
|---|---|---|---|
| **Web应用** | 前后端分离、首屏渲染、CDN、XSS防护 | “帮我设计一个高并发Web应用的SSR架构和自动化测试方案” | `sdlc-dev` + `sdlc-test` |
| **移动应用** | 多端适配、弱网优化、热修复、App商店分发 | “移动端App如何做自动打包流水线和崩溃率监控？” | `sdlc-ci-cd` + `sdlc-ops` |
| **大数据系统** | 流批一体、数据血缘、列存储、资源隔离 | “请提供大数据流处理组件的技术选型与容灾压测方案” | `sdlc-dev` + `sdlc-ops` |
| **AI平台** | 模型管理、推理延迟、GPU调度、数据脱敏 | “设计一个AI大模型推理服务的容器化部署和蓝绿发布流程” | `sdlc-ci-cd` + `sdlc-ops` |

### 维度四：按合规要求 (Compliance)
| 合规标准 | 强制性约束 | 典型调用示例 | 触发技能 |

## 🚀 组合调用实战 (使用公式)

您可以直接将上述维度的关键词组合起来，作为 Prompt 发送给智能助手：

> **公式**: `[触发场景/任务] + [维度一：项目类型] + [维度二：技术栈] + [维度三：团队规模] + [维度四：合规要求]`

**实战案例 1 (构建一个完整的起步规范):**
> “我们是一个**10人左右的初创团队**，正在研发一款**面向欧洲用户的Web电商应用 (GDPR)**，后端使用**Go**，前端使用**React (Node.js)**。请帮我调用 `sdlc-dev` 和 `sdlc-ci-cd` 技能，输出适合我们的**架构设计建议**、**前后端代码规范**以及**基于GitHub Actions的CI/CD流水线设计**。”

**实战案例 2 (排查与治理遗留系统):**
> “我们是一个**大型金融组织 (等保2.0, SOX)**，目前在使用**Java/Spring Cloud**构建复杂的**微服务大数据系统**。请帮我调用 `sdlc-test` 和 `sdlc-ops`，制定一套符合审计要求的**全链路自动化压测方案**、**灰度发布策略**以及**容灾演练的交付物模板**。”

通过这套矩阵，您可以随时、随地、按需获取最匹配当前项目上下文的工程最佳实践！
